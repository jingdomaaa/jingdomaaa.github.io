图床的选择与使用

jingdom 2020-12-15 图床

# 图床的选择与使用

# 前言

在做个人博客网站的时候发现图片使用路径方式引用时，上传到网络上时无法显示

稍加考虑自然是因为路径发生改变

在观察其他网站的图片路径后发现

网站的图片路径是使用网址来引用的

于是图床的概念就出现了——储存图片的服务器

# 图床的选择

# GitHub

一开始我对图床概念不了解，仅仅根据我需要的功能去在我知道的工具中寻找，发现GitHub可以储存图片，于是在开始阶段我的图片都是由我上传到GitHub中再使用的

可通过一段时间的使用之后发现，以GitHub作为图床加载图片必须翻墙，不然很大概率加载不出，这与初衷相悖，所以便开始了我的图床选择之路

# 阿里云OSS+PicGo

几乎搜集了网上所有的图床推荐以及使用方案，最终选择了阿里云，因为相对安全、稳定。

  • 应该是我找到的最全的图床评价

    • markdown图床-简书

  • 该文章推荐使用码云作为图床，并以uPic作为图床管理工具

  • 优点：

    • 免费
    • 国内访问速度较快

  • 缺点：

    • 当图片大于1M时需要登录才能查看，不过一般情况下用于博客的图片是小于1M的
    • 虽然目前码云还没有针对将仓库作为图床的行为有较大的限制，不过可能在将来会把图片访问加以限制

其他免费的图床也都是免不了以上各种问题，尤其是不知道图片上传之后网站什么时候会挂掉。

所以综上，我选择了阿里云OSS，最近双十二很便宜，一年8.4元，并且由于是大厂，安全有保障

当然为了保证图片在自己这里有一个备份，最好在本地、百度云、迅雷等云盘上保存自己的图片

# 阿里云OSS购买与配置

# 阿里云OSS购买选择

最基本的图床配置需要支付储存费用与流量费用

具体请看阿里云对象存储OSS定价

储存费用：

  • 储存费用指的是：
    • 阿里云帮你储存文件，代价是需要支付阿里一定量的储存费用

我选择的是标准型LRS储存包40GB，价格是8.4元/年，相比按照单价购买便宜很多

储存费用计费详情：

流量费用：

  • 流量费用指的是：
    • 从网络下行文件会产生流量消耗，故需要支付阿里流量费用
    • 假如你的OSS需要访问量很大会造成较大的下行流量，推荐购买下行流量包，由于我个人观看博客使用，故目前没有购买

流量费用计费详情（以华东1为例）

其他收费之后用到的时候再进行解释

当你完成购买之后需要对OSS进行一番配置

# 阿里云OSS配置

# 在阿里云创建Bucket

进入阿里云控制台，如果没有开通对象存储，需要先进行开通 （开通不收费，使用才收费）

进入对象存储的配置中，选择Bucket列表 -> 创建Bucket，配置调整如下：（读写权限 需要调整 公共读）

# 获取AccessKey SecretKey

上传图片 需要这两个参数，建议使用RAM子账号，授权OSS，不建议使用主账号的，风险大

进入 AccessKey管理。进入之后，会弹出一个对话框，选择开始使用子用户AccessKey

创建子用户，选择编程访问，点击确定，输入手机验证码验证身份

创建成功后，就会出现需要的信息，这里的信息只会显示一次，自己备份下来

返回到用户列表 ，找到刚才创建的子用户，给他添加一个OSS的权限

这里我们选择 FullAccess

至此，阿里云OSS这边的准备工作就完成了

# 使用PicGo实现快速上传图片

PicGo是一个开源项目，可以用来快捷上传图片到图床。

下载picGo

其中有很多版本可供选择，这里选择了最新的稳定版

下载完exe文件之后双击安装，可以自定义安装位置

打开PicGo，进行图床设置

  • accesskey和accesskeySecret在之前创建子用户时要求保存的信息里面
  • 存储空间名是bucket的名字
  • 存储区域到阿里云OSS控制台去找，例如下图中存储区域就是oss-cn-hangzhou

  • 存储路径是存储图片的位置，要求以/结尾
  • 自定义域名可以不填写

至此，PicGo配置完成

# 开始上传

打开PicGo

配置了之后，在截图了之后，可以点击剪贴板图片上传，也可以使用快捷键快速上传剪贴板图片（快捷键在PICGO设置中可以设置）

上传之后，会自动拷贝链接格式，粘帖得到的结果是链接格式。上面我们选择的是markdown，所以粘帖结果会是markdown引用图床图片的结果

图床搭建完成
